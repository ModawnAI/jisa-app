# 가격 모델 요약 (빠른 참조용)

## 핵심 원칙

```
1. 모든 사용자 = 동일 요금 (₩30,000/사용자/월)
2. 티어 = 지식 접근 레벨 (기능 차이 없음)
3. 청구 = 활성 사용자 수 기준
4. B2B 모델 = 기업당 월간 구독
```

---

## 티어 vs 역할

### 티어 (지식 접근 레벨)

```
무료 ────────► 공개 정보만
베이직 ───────► 기본 내부 정보
프로 ────────► 전문 지식
엔터프라이즈 ──► 모든 정보 (경영 데이터 포함)

모든 티어 요금: ₩30,000/월
```

### 역할 (시스템 권한)

```
사용자 ───────► 기본 쿼리
주니어 ───────► 기본 대시보드
시니어 ───────► 고급 검색
관리자 ───────► 팀 관리
총괄 관리자 ───► 부서 관리, 사용자 관리
대표이사 ─────► 모든 권한
```

---

## 청구 계산식

```
월간 청구액 = 활성 사용자 수 × ₩30,000 × (1 - 할인율)

예시:
- 50명 활성: 50 × 30,000 = ₩1,500,000
- 볼륨 할인 5%: 1,500,000 × 0.95 = ₩1,425,000
```

## UI 변경사항

### Before (혼란스러움)
```
구독 티어: [무료 v] [베이직] [프로] [엔터프라이즈]
```

### After (명확함)
```
지식 접근 레벨: [무료 (공개 정보만) v]
                [베이직 (기본 내부 정보)]
                [프로 (전문 지식)]
                [엔터프라이즈 (모든 정보)]

ℹ️ 모든 레벨 동일 요금. 레벨은 접근 가능한 정보의 범위를 결정합니다.
```

## 빠른 FAQ

**Q: 엔터프라이즈 티어가 더 비싼가요?**
A: 아니요. 모든 티어 ₩30,000/월로 동일합니다.

**Q: 티어를 변경하면 요금이 바뀌나요?**
A: 아니요. 요금은 변하지 않고 접근 가능한 정보만 바뀝니다.

**Q: 왜 티어가 있나요?**
A: RBAC 콘텐츠 필터링을 위한 것입니다. 직원마다 필요한 정보 수준이 다르기 때문입니다.

**Q: 청구는 어떻게 되나요?**
A: 월말 기준 활성 사용자 수 × ₩30,000/월입니다.

**Q: 비활성 사용자는?**
A: 해당월 쿼리가 없으면 청구하지 않습니다.
